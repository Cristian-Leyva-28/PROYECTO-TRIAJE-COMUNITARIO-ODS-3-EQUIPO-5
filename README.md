PROBLEMATICA: 
En una brigada comunitaria se clasifican pacientes con prioridades: Emergencia, Urgente, No urgente, Control. El sistema debe registrar casos por hora (bloques), mostrar porcentajes, tiempo de atención estimado y emitir alertas si la proporción de emergencias supera un umbral.

PROPOSITO Y FUNCIONAMIENTO: 
ESTE PROGRAMA FUE CREADO CON EL LENGUAJE C Y EN EL COMPILADOR DE DEV-C++. AL INICIAR EL PROGRAMA SE MOSTRARA EL MENU CON LAS OPCIONES QUE HAY PARA SALECCIONAR, CADA OPCION ES MANEJADA POR DIFERENTES FUNCIONES QUE TIENEN PROPOSITOS DIFERENTES. A CONTINUACION SE EXPLICARA LA DIVISION DE TAREAS DE CADA FUNCION.

REQUISITOS PARA CORRER EL PROGRAMA:
USAR EL COMPILADOR DEV C/C++ Y USAR SISTEMA OPERATIVO WINDOWS

DIVISON DE TAREAS:
EN TOTAL SE USARON EN TOTAL 7 FUNCIONES

 °FUNCION 1: LIMPIARBUFFER()
 
   ESTA FUNCION SIRVE PARA ARREGLAR PROBLEMAS DE BUGS EN EL PROGRAMA. EN EL PROGRAMA PIDE QUE EL USUARIO ELIJA UNA VARIEDAD DE OPCIONES LAS CUALES SE REPRESENTAN MEDIANTE NUMEROS
   Y POR ESE MOTIVO USAMOS VARIABLES TIPO ENTERO, PERO SI EL USUARIO PONE UNA LETRA O SIMBOLO, EL PROGRAMA SE ROMPE. ¿COMO SE SOLUCIONO? USANDO UN FFLUSH(STDIN) QUE OBLIGA AL PROGRAMA 
   A QUE BORRE EL CONTENIDO DEL BUFFER Y LA LETRA O SIMBOLO QUE ESCRIBA EL USUARIO NO SE QUEDE GUARDADA EN LA VARIABLE DE OPCION Y ROMPA EL PROGRAMA.

   SE CREO UNA VARIABLE Y UTILIZAMOS UNA CONDICION ( UN IF ) EL CUAL DICE QUE SI ESA VARIABLE ES DIFERENTE A 1 ENTONCES LIMPIE EL BUFFER. LA VARIABLE SE LLAMA RESULTADO
   Y CADA VES QUE SE USA UN SCANF DECIMOS QUE RESULTADO SERA IGUAL AL VAOR QUE SCANF ARROJE (RECORDEMOS QUE EL SCANF TIRA UN UN NUMERO DE CARACTERES QUE PUEDO LEER PERO SOLO
   PODRA LEER LOS CARACTERES QUE SE LE INDIQUE, EN ESTE CASO SOLO VA A LEER NUMEROS ENTEROS) Y SI EL SCANF NO LEYO NINGUN CARACTER (POR CULPA DE LAS LETRAS O SIMBOLOS) ENTONCES
   SE LLAMARA LA FUNCION DE LIMPIAR BUFFER Y SE EVITARAN ERRORES EN EL PROGRAMA.


 °FUNCION 2: UMBRAL DE EMERGENCIAS
   ESTA FUNCION SIRVE PARA MOSTRAR EN PANTALLA TODOS LOS CASOS DE EMERGENCIAS QUE HAY REGISTRADOS, SIRVE PARA EMITIR UNA ALTERTA CUANDO EL UMBRAL SUPERE LOS 10 CASOS REGISTRADOS.
   ¿COMO SE LOGRA ESTO? USAMOS UN ACUMULADOR QUE CUANDO EL USUARIO INGRESE UN CASO DE EMERGENCIA EL ACUMULADOR VAYA INCREMENTANDO Y VAYA EN SINCRONIA CON EL NUMERO TOTAL DE CASOS DE
   EMERGENCIA.
   SOLAMENTE SE UTILIZO UN IF Y UN ELSE PARA QUE DEPENDIENDO DEL NUMERO DE CASOS MUESTRE UN MENSAJE DE LA CANTIDAD DE CASOS O MUESTRE UNA ALERTA DE NUMERO DE CASOS SUPERADOS.
   ESTA FUNCION SERA LLAMADA EN TODA OPCION DEL PROGRAMA.

  °FUNCION 3: AGREGAR CASO COMUNITARIO
   ESTA FUNCION ES UN POCO EXTENSA PERO LO QUE HACE ESTA FUNCION ES QUE EL USUARIO PUEDA ELEGIR LA PRIORIDAD DEL CASO, ESTAS PRIORIDADES SON EMERGENCIA, URGENTE, NO URGENTE Y CONTROL
   EL USUARIO ELIGIRA SU OPCION Y SE GUARDARA EN UNA VARIABLE LLAMADA "TIPO" EL CUAL ES UNA VARIABLE GLOBAL, DESPUES SE LE DARA LA OPCION DE DECIDIR CUANTOS REPORTES HARA EN ESE MOMENTO,
   EL LIMITE DE REPORTES ES DE 5, ESTO SE HIZO ASI PARA EVITAR QUE EL USUARIO PONGA UNA CANTIDAD EXAGERADA DE REPORTES Y CAUSE PROBLEMAS AL PROGRAMA, LA CANTIDAD DE REPORTES ESCRITA
   SE GUARDARA EN UNA VARIABLE LLAMADA "CANTIDAD REPORTES", DESPUES DE ESO EL USUARIO VA A REGISTRAR LA HORA DEL CASO, LA HORA QUE DEBERA REGISTRAR SERA DEL FORMATO DE 24 HORAS, SOLO
   PODRA INGRESAR LA HORA, NO MINUTOS. ESTA INFORMACION SERA GUARDADA EN UNA VARIABLE LLAMADA "HORA".
   ESTAS VARIABLES AYUDARAN PARA PODER GUARDAR LA CANTIDAD DE CASOS EN NUESTRO ARREGLO DE "CASOS[4][24]"
   DESPUES DE ESO DEPENDIENDO DE LA PRIORIDAD QUE HAYA ELEGIDO EL USUARIO SE MOSTRARA EN PANTALLA EL TIEMPO DE ESPERA QUE TENDRA EL CASO.
   EN CASO DE SER UNA PRIORIDAD EMERGENTE SE MOSTRARA QUE SE DEBE ATENDER DE MANERA INMEDIATA Y SE LE AUMENTARA LA CANTIDAD DE REPORTES AL ACUMULADOR DE EMERGENCIAS (VARIABLE USADA
   EN LA FUNCION 1).
   AL FINAL DEL LA FUNCION TENDRAS QUE PRESIONAR DODBLE ESPACIO PARA VOLVER AL MENU, ESTO SE HACE CON DOS GETCHAR, UNO PARA BORRAR LO QUE HAY EN EL BUFFER Y OTRO PARA RECIBIR EL ENTER

 °FUNCION 4: MOSTRAR TABLA
  ESTA FUNCION SIRVE PARA MOSTRAR UNA TABLA CON TODOS LOS CASOS REGISTRADOS EN LAS 24 HORAS. ESTO SE LOGRA USANDO LAS VARIABLES UNIVERSALES LLAMADAS "TIPO Y HORA" Y TAMBIEN DOS
  CICLOS FOR. UTILIZANDO LAS VARIABLES DE TIPO Y HORA COMO INDICES PODREMOS USARLOS EN LOS FOR PARA PODER IMPRIMIR LOS DATOS QUE HAY GUARDADO EN EL ARREGLO DE CASOS[4][24],
  CABE RECALCAR QUE ESTE ARREGLO ES BIDIMENSIONAL.
  AL FINAL DEL LA FUNCION TENDRAS QUE PRESIONAR DODBLE ESPACIO PARA VOLVER AL MENU, ESTO SE HACE CON DOS GETCHAR, UNO PARA BORRAR LO QUE HAY EN EL BUFFER Y OTRO PARA RECIBIR EL ENTER

 °FUNCION 5: MOSTRAR ESTADISTICAS:
  ESTA FUNCION MOSTRARA EN PANTALLA LA CANTIDAD DE CASOS DE CADA PRIORIDAD QUE HUBO EN EL DIA, POR EJEMPLO: "LA PRIORIDAD DE EMERGENCIA TUVO 6 CASOS REGISTRADOS EN TODO EL DIA Y TAMBIEN   LO MOSTRARA REPRESETADO EN PORCENTAJE, 
  LA PRIORIDAD DE URGENCIA TUVO 2 CASOS REGISTRADOS EN TODO EL DIA Y REPRESENTADO EN PORCENTAJE ES 50%...", TAMBIEN MOSTRARA LA HORA O LAS HORAS CON MAS CASOS QUE HUBO EN CADA PRIORIDAD   DURANTE TODO EL DIA, POR EJEMPLO:
  "EN EMERGENCIA LA HORA CON MAS CASOS FUE A LAS 2 Y 3 HRS" SOLO SE MOSTRARA EN PANTALLA MAS DE UNA HORA CUANDO HAYAN UNA CANTIDAD DE CASOS IGUALES (A LAS 2 HUBO 5 CASOS Y A LAS 3 HUBO 5
  CASOS), TAMBIEN SE MOSTRARA UNA SUMA TOTAL DE TODOS LOS CASOS DE TODAS LAS PRIORIDADES QUE HUBIERON DURANTE EL DIA Y SERA REPRESENTADO 
  COMO PORCENTAJE.

  PARA HACER LA SUMATORIA DE TODOS LOS CASOS QUE HUBIERON EN CADA PRIORIDAD TUVIMOS QUE HACER UNA SUMATORIA DE TODOS LOS CASOS DE CADA PRIORIDAD, ESTO SE HIZO CON UN FOR Y CON 4 ACU-
  MULADORES (VARIABLES LOCALES DENTRO DE ESTA FUNCION) Y DESPUES CON VARIOS IF Y ELSE MOSTRARIA EN PANTALLA LA SUMATORIA TOTAL DE CADA PRIORIDAD PERO EN EL CASO DE NO HABER CASOS 
  REGISTRADOS SE MOSTRARIA EN PANTALLA QUE NO HAY CASOS REGISTRADOS.

  PARA HACER LA HORA CON MAS CASOS DE CADA PRIORIDAD SE TUVO QUE UTILIZAR UN FOR Y 4 IF PARA IR COMPARANDO CUALES ERAN LOS NUMEROS MAS GRANDES DE CADA PRIORIDAD, EL NUMERO MAS GRANDE
  SE GUARDARIA EN UN ACUMULADOR/VARIABLE (CADA PRIORIDAD TENDRIA SU PROPIO ACUMULADOR, POR ENDE SON 4 ACUMULADORES USADAS).
  DESPUES DE ESO CON OTRO FOR SE BUSCARIA LA HORA O LAS HORAS EN LA QUE HUBO ESA CANTIDAD DE CASOS (AQUI SE USAN LOS ACUMULADORES) Y DENTRO DE ESE FOR IRIAN 4 IF Y ELSE EN DONDE SI NO 
  HAY CASOS REGISTRADOS SE MOSTRARIA EN PANTALLA O SI HAY CASOS REGISTRADOS PUES MOSTRARIA LA HORA O LAS HORAS EN DONDE HUBIERON ESA CANTIDAD DE CASOS.

  PARA HACER UNA SUMA TOTAL DE TODOS LOS CASOS SE UTILIZO DOS VARIABLES LLAMADAS "INDICE 1 E INDICE 2", FUERON UTILIZADAS EN DOS FOR PARA IR SUMANDO TODOS LOS CASOS DE CADA PRIORIDAD
  EN TODO EL DIA, AQUI SE UTILIZO OTRO ACUMULADOR LLAMADO "CASOS TOTALES" Y DESPUES DE ESO CON UN PRINTF SE MOSTRARIA LA CANTIDAD DE CASOS TOTALES

 °FUNCION 6: RECOMENDACIONES:
  UNA FUNCION DONDE HAY SOLAMENTE TEXTO (PRINTF) EN DONDE LE EXPLICA AL USUARIO EL FUNCIONAMIENTO DE CADA OPCION Y LE DA RECOMENDACIONES DE COMO TRABAJAR 
  AL FINAL DEL LA FUNCION TENDRAS QUE PRESIONAR DODBLE ESPACIO PARA VOLVER AL MENU, ESTO SE HACE CON DOS GETCHAR, UNO PARA BORRAR LO QUE HAY EN EL BUFFER Y OTRO PARA RECIBIR EL ENTER.

 °FUNCION 7: RESETEAR DATOS:
  ESTA FUNCION USA DOS CICLOS FOR Y DOS INDICES PARA PODER IGUALAR TODOS LOS DATOS DEL ARREGLO "CASOS[4][24]" A 0, ESTA FUNCION SIRVE PARA RESTABLECER TODOS LOS DATOS A 0,
  ES UNA SIMULACION DE CUANDO PASAN LAS 24 HORAS Y LOS DATOS DEBEN VOLVER A 0
  AL FINAL DEL LA FUNCION TENDRAS QUE PRESIONAR DODBLE ESPACIO PARA VOLVER AL MENU, ESTO SE HACE CON DOS GETCHAR, UNO PARA BORRAR LO QUE HAY EN EL BUFFER Y OTRO PARA RECIBIR EL ENTER.

 °FUNCION 8: BORRAR CASO COMUNITARIO
  FUNCIONA CASI IGUAL QUE AGREGAR CASO COMUNITARIO- ELIGE LA PRIORIDAD, ELIGE CUANTOS CASOS DESEAS BORRAR, ESTA OPCION SOLO PUEDES BORRAR 5 CASOS PERO HAY UNA CONDICION LA CUAL
  UTILIZAMOS
  UN IF Y UN ELSE. LA CONDICION DICE QUE SI EL USUARIO INTENTA BORRAR UNA CANTIDAD MUCHO MAYOR A LA CANTIDAD DE CASOS QUE HAY REGISTRADOS EL PROGRAMA TE DIRA QUE NO ES POSIBLE Y TE 
  MANDARA AL MENU PRINCIPAL, EN EL CASO DE QUE QUERAMOS RESTARLE UNA CANTIDAD MUCHO MENOR A LA CANTIDAD DE CASOS REGISTRADOS PUES ENTONCES ESTA ACCION SE CUMPLIRA Y SE ELIMIARA LA 
  CANTIDAD DE CASOS QUE EL USUARIO HAYA ELEGIDO INCLUYENDO SI LA PRIORIDAD ES LA NUMERO 1 (EMERGENCIA) TAMBIEN AL ACUMULADOR DE EMERGENCIAS (UMBRAL) SE LE RESTARA LA CANTIDAD ELEGIDA
  ANTERIORMENTE. LA PRIMERA CONDICION SE HIZO PARA EVITAR QUE LOS CASOS SE QUEDARAN EN CASOS NEGATIVOS.
  SI EL USUARIO QUIERE BORRAR ALGÚN 
  CASO COMUNITARIO PERO EL ARREGLO
  ESTA VACIO EL PROGRAMA LE DIRA 
  QUE NO PUEDE HACERLO A MENOS
  QUE HAYA REGISTRADO UN CASO.
  AL FINAL DEL LA FUNCION TENDRAS QUE PRESIONAR DODBLE ESPACIO PARA VOLVER AL MENU, ESTO SE HACE CON DOS GETCHAR, UNO PARA BORRAR LO QUE HAY EN EL BUFFER Y OTRO PARA RECIBIR EL ENTER.

 °FUNCION INT MAIN:
  MOSTRARA EL MENU PRINCCIPAL CON 7 OPCIONES: AGREGAR CASO, MOSTRAR TABLA, MOSTRAR ESTADISTICAS, RECOMENDACIONES, RESETEAR CASOS, BORRAR CASO Y SALIR DEL PROGRAMA.
  DEPENDIENDO DE LA OPCION SE LLAMARA UNA FUNCION DIFERENTE

ESTOS FUERON EN TOTAL LA DIVISON DE TAREAS Y LA CANTIDAD DE FUNCIONES UTILIZADAS.
SE UTILIZARON 8 VARIABLES GLOBALES, 12 VARIABLES LOCALES EN LA FUNCION 5 DE MOSTRAR ESTADISTICAD, 1 VARIABLES LOCALES EN LA FUNCION 7 RESETEAR CASO, 1 VARIABLE LOCAL EN LA FUNCION
8 BORRAR CASO Y 1 VARIABLE LOCAL EN LA FUNCION MAIN.
